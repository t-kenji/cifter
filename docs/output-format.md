# 出力フォーマット

`cifter` の公開出力は行番号付き text のみです。JSON や構造化 API は v0 の対象外です。

## 基本契約

- すべての出力行は `<line_no>: <text>` 形式です
- 行番号は 1-based です
- `text` は元ソースの該当行をそのまま保持します
- 出力順は元ソースの行順です
- 色付き出力時も可視文字列としての `<line_no>: <text>` 契約は変わりません

## 表示規則

- 桁幅は最後の出力行番号に合わせて右寄せします
- 行内容のインデントや空白は保持します
- 前処理で空行化された行は、抽出対象に残らない限り出力しません
- 色付き出力はコード部分に ANSI エスケープを加えます
- 行番号プレフィクスの書式は色付きでも維持します
- 色付き出力では行番号プレフィクスを視認性調整のため弱表示します
- `flow --highlight` の追加強調位置は、タブを 4 桁幅で展開した後の表示列に合わせます

## トレーサビリティ

- `ExtractionResult.span` は元ファイルと開始行・終了行を保持します
- `lines` は個々の表示行を `ExtractedLine(line_no, text)` で保持します
- `function` / `flow` / `path` のいずれも、元ソースへの対応関係を失わないことを優先します
